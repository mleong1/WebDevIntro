    Important points from HTML course.

    • Tags are not displayed by the browser but tell the browser how to process content or what they represent.
    • HTML document is a collection of nested HTML tags that describe how the browser should read the content. Handles structure of the document.
    • Head element handles the words inside of the tab in the browser (Probably where favicons go too).
    • Doctype confirms that the document follows HTML5.
    • Attribute is an extra piece of information in a tag to give it definition, context, or identity. EX. id =
    • Formatting tags include bold, strong, italics, emphasis, superscript (ex. n2), and subscript (ex. H2O).
    • Inserted text? and deleted text and marked text.
    • public static void main(String args[]){
      System.out.println("Hello world!")
      }
    • Link to code along 2.
    • A website's homepage can have one of three filenames: index.html, home.html, or default.html.
    • Block elements create a block of content in its parent started on a newline. Paragraphs, divs, head, body, etc.
    • Content to be seen in the browser must be in the body.SS
    • Semantic elements (footer, header, article, section, main) are just divs that are named so that you can organize css and js better and also help organize the items.
    • HTML code characters always start with the & (ampersand) symbol and end with ; (semi-colon). ©
    • Inline element must be nested in a block element and does not take up horizontal width of parent container. Span is the inline Div.
    • Anchor elements identifies content that a user can click to open a separate page.
    • Target attribute for anchor tag defines how the linked content will open. This is how you have an anchor tag open to a new tab.
    • Internal links use relative paths to the file you are linking to.
    • Specifying only height or width resizes the image while maintaining the aspect ratio.
    • Definition list is the 3rd list I had never heard of. DT to create thing to be defined and DD for the definition.
    • Input tag is very versatile and can take many forms based off of the attribute tag like text, radio or submit.
    • Some HTML elements have different states (when you hover over a link). Pseudo classes let you style those states.
    • For links and their pseudo classes, they should be styled in the order of link, visited, hover, active.

    Important points from the CSS course

    • While HTML handles structure of document, CSS handles how it looks and is laid out.
    • CSS rule set is the block of code that handles all of the styling for an HTML element or group of elements.
    • CSS selectors: (. for class, # for id, and nothing for an existing html element.
    • CSS declarations end in semi-colons ;
    • CSS declarations are made of of properties and values. Name the property and then the value that you want the property set to. Color: red;
    • Same comments as Java, /* */
    • Applying styling to multiple elements is very DRY so we don't repeat code everywhere and then have to clean it up if it goes wrong.
    • Formatting applied to parent elements is applied to children and nested elements. Inline elements in a paragraph for instance.
    • You can list elements by comma separating them so that CSS styling is applied to multiple selected elements.
    • Inline styling is when you use the style attribute of a HTML element. Internal style sheet is a set of rules applied to the head element (page specific). External style sheet is a separate .css file used to format a page applied via style link.
    • Inline styles take precedence over the other methods.
    • Padding is whitespace around the content in the border, the border is the literal border around the content, and the margin is the whitespace around the border.
    • Fixed values are a measurement of pixels. Relative values are percentages of parent elements.
    • Padding takes fixed lengths. There is a padding-top, padding-left, padding-right, and a padding-bottom. You can also use padding: 5px, 15px, 15px, 5px to set top, right, bottom left. Or you can use padding: 5px to set all sides.
    • Border also lets you specify all 4 directions. You can specify a thickness, style, and a color.
    • Margin is the same as padding just outside of the border.
    • Formula for the height of an element: margin-top + border-top + padding-top + (height of the content) + padding-bottom + border-bottom + margin-bottom
    • Formula for the width: margin-left + border-left + padding-left + (width of the content) + padding-right + border-right + margin-right
    • Keep contrast up between font and background color. Light font on a dark background or vice versa.
    • Background shorthand: background: color image repeat attachment position;
    • Color specifically only changes text color.
    • The CSS attribute text-align allows us to align text to the left of the page, to the right, centered, or justifed.
    • If you use fixed sizes like pixels (px) or points (pt) instead, you run the risk of having very large fonts on small screens or very small fonts on larger screens.
    • If you include two selectors separated by a space (instead of a comma), the first selector is the parent of the second selector, and the style will apply only to children of that parent. In this case, the notation tfoot td means "td elements inside a tfoot element," and the style will not apply to other td elements in the table.
    • Thead contains the head row and TBody contains the body rows.
    • Caption needs to be in the table element.
    • Margin: auto will center the block.
    • It is common to use a width between 900 and 1200 pixels.
    • We can turn off the float setting using the declaration clear:both, which tells the browser to turn off any floats (left or right) that are currently in effect.
    • Paragraph are easier to wrap around floated elements than lists or tables.
    • Bootstrap is a CSS framework that has already built out CSS structures for usage. Built on jQuery.js and popper.js.
    • Bootstrap is probably so popular because it prioritizes mobile first development since we browse so much on smartphones today.
    • CDN stands for Content Delivery Network, and it is a widely-used option for incorporating a framework like jQuery or Bootstrap into a website.
    • In order for Bootstrap JavaScript to work, you must add jQuery and popper.js to your website, BEFORE adding Bootstrap. This is most likely because bootstrap calls code in those JS frameworks.
    • Meta tag is used to provide info on the webpage. Charset tells us the character set used on the page. Viewport tells the browser to look at the user's window to determine how to apply Bootstraps media queries. Tags added to this sample page.

    Important points from Bootstrap section of the course.

    • Adding bootstrap via a CDN.
    • Common practice to add JS files to just before the /body tag so that the user can see and use the HTML content while the browser is loading the JS in the background.
    • Responsive design is to account for your user's use cases which means using programming to handle when a use case arises. Design for all contingencies and browsers. This often is talking about resizing based on browser (mobile, web).
    • To be mobile first, we need to leverage Bootstraps container class divs and its 12 column grid system.
    • Media queries are used in CSS to apply styles based on conditions like size of the viewport or device, orientation (landscape, portrait mode) or type of screen.
    • Container vs container fluid is what is going to control whether the cells stack when the page is resized or if the cells just get more narrow in the grid system.
    • Only columns may be immediate children of rows.
    • Grid breakpoints are based on minimum width media queries, so they apply to that breakpoint and all of those above it. So if you just specify col-md-4, it will allot that space for the large breakpoint.
    • Bootstrap rewrites the default font for headings, text, inline elements, and lists.
    • Bootstrap makes tables easier with built in classes for alternating row styling, fixing borders, or adding color to tr and td elements.
    • You can make images response via bootstrap. (.img-fluid)
